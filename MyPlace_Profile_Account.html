Sub-Screen Specs
Key design decisions and component specs for all 6 sub-screens

Edit Profile

Inline editing · Field-level edit/save

Components
Top Bar — Save button (right) Inline Field — Static state Inline Field — Active/editing state Warning — Email re-verification notice
Inline Edit Specs
Static stateWhite bg · gray-100 border · "Edit" link top-right of label
Active statepurple-500 border · purple-50 glow ring · Cursor visible
Multiple editsMultiple fields can be active simultaneously
Save buttonIn top bar (right) · Active when any field changed · "Saving…" on submit
Email changeWarning notice appears · Requires re-verification after save
CancelReverts all unsaved changes · Confirms if changes pending
Navigation
Edit Profile→Save→Profile Hub (updated)
Edit Profile→Email changed + save→Verify Contact

Address Management

View, add, edit, delete, set primary

Components
Top Bar — "+ Add" button (right) Primary Address Card (purple tint) Secondary Address Card (white) Add New Address CTA (dashed border)
Address Card Specs
Primarypurple-50 bg · purple-100 border · "Primary Delivery" label
SecondaryWhite bg · gray-100 border · Custom label (Home, Work, etc.)
Actions rowEdit · Set as Primary · Delete — shown below address text
DeleteConfirmation modal before delete · Cannot delete primary if only 1 address
Set as primarySwaps current primary · Updates subscription delivery address
Add newDashed border card → navigates to Address Entry form
Navigation
Addresses→Edit address→Address Entry (pre-filled)
Addresses→Add new→Address Entry (blank)

Notification Preferences

Toggle-based · Auto-saves on toggle

Notification Types
Delivery UpdatesDefault ON · Shipping + arrival alerts
Order ConfirmedDefault ON · Sent when order placed
Exclusive OffersDefault ON · Allstate-exclusive deals
Renewal RemindersDefault OFF · Pre-policy renewal nudge
Marketing EmailsDefault OFF · Inherits from T&C opt-in
Toggle Specs
On statepurple-500 bg · Dot right
Off stategray-200 bg · Dot left
Save behaviorExplicit "Save Preferences" button · Prevents accidental changes

Privacy Settings

Data controls + account deletion

Settings
Usage AnalyticsDefault ON · Anonymous app usage data
PersonalizationDefault ON · Product suggestions
Download DataSends data export to registered email · GDPR compliance
Delete AccountRed styling · Multi-step confirmation · Irreversible · 30-day grace period
💡 Delete Account must have a multi-step confirmation flow (at minimum: "Are you sure?" → "Type DELETE to confirm"). Add a 30-day recovery window before permanent deletion.

Change Password

3-field form + strength indicator

Fields & Validation
Current passwordRequired · Verified against backend before accepting new
New passwordMin 8 chars · Strength indicator below field
ConfirmMust match new · Error shown inline on mismatch
Strength bar4 segments · Colors: red / orange / gold / green · "Weak / Fair / Good / Strong" label
Show/hideEye icon on all 3 fields
Navigation
Change Password→Success→Profile Hub + success toast
Change Password→Wrong current pw→Inline error on current field

Linked Allstate Account

Read-only policy details + re-verify option

Fields Shown
Policy TypeRenters / Homeowners / Auto
Policy NumberMasked partially · Full number shown
Coverage AddressAddress on file with Allstate
Renewal DateFrom policy data · Highlights if within 60 days
Verified OnDate of last MyPlace verification
Actions
Re-verifyTriggers Policy Entry flow again · Useful after policy renewal
Contact AllstateOpens Allstate support URL in browser
Header styleAllstate blue gradient header (not purple) · Brand differentiation
💡 The Allstate card header uses Allstate blue — not MyPlace purple. This is intentional. It visually signals "this is Allstate data, not MyPlace data" and reinforces the partnership without conflating the two brands.
