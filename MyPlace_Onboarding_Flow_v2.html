Screen 8 — Location Permissions

Single purpose · Two paths to same outcome (map works either way)

Components Used
Top Bar — Logo only (no back) Button — Primary Brand (Allow Location) Divider with label Input — ZIP code (centered, large) Button — Primary Blue (Use This ZIP) Micro copy — privacy reassurance
Layout Specs
LayoutCentered · Icon + title + body · Two options separated by divider
Location icon48px emoji or custom SVG · Top of content area
ZIP input styleCentered text · 18px · letter-spacing 4px · Numeric keypad
Privacy note10px · gray-300 · Intentionally subtle
Permission Flow
Allow tappedTriggers native browser location prompt · If granted → map uses GPS · If denied → falls back to ZIP input
ZIP submittedValidates 5-digit format · Stores as user preference · Map centers on ZIP
Skip / laterNot offered — location is required for store map. ZIP is the fallback.
Navigation
Location→Allow or ZIP submitted→Map Flow (Store Selection)
Map Flow→Store confirmed→Address Entry
💡 No back arrow on this screen — users can't skip location/ZIP. It's required for the store map. The ZIP fallback ensures 100% of users can proceed.
