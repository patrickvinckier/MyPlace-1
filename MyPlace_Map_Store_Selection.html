Entry points: Onboarding → Location Permissions Home Pre → "Change Store" Subscription Registration → Store step Profile → Address Management (future)

Screen 1 — Map View

Full-screen map with overlay controls · Bottom sheet store list

Components Used
Top Bar — Back + Title + Menu Search Bar Overlay Filter Chips Row (5 options) Location Refresh Button Map — Full Screen (Google Maps embed) Map Pins — Selected (purple) + Unselected (white) User Location Dot Bottom Sheet — Store List Store List Cards (scrollable)
Map Layout Specs
Map heightFull screen minus top bar · ~340px on standard mobile
Map libraryGoogle Maps JS API · or Mapbox as alternative
Search barFloating overlay · top 12px · white bg · shadow-lg
Filter rowHorizontal scroll · Below search bar · shadow-md chips
Active filterpurple-700 bg · white text · Filters map pins in real time
Location btnTop-right · 40px circle · White · Recenters map on user
Map Pin Specs
Selected pin36px circle · grad-purple · white border · Label below with purple-700 bg
Unselected pin36px circle · white bg · gray-200 border · Gray label
User dot14px · blue-500 · White border · Pulsing blue glow ring
Pin tapPin → selected state · Bottom sheet scrolls to that store card
Store card tapMap pans + zooms to pin · Pin → selected state
Bottom Sheet Specs
Default statePeek height ~200px · Shows 2–3 cards · Drag handle visible
Expanded stateFull height · All stores visible · Map shrinks behind
Handle36px × 4px · gray-200 · Centered · Drag to expand
Selected cardpurple-50 bg · Left border 3px purple-500
Open/closedStatus color: success green / error red
Closed storeStill shown · Opening time displayed · Can still be selected
Search Bar Behavior
Input typeAddress or ZIP code · Geocodes and recenters map
AutocompleteGoogle Places autocomplete · Filters to US addresses
Clear button✕ · Visible when text entered · Clears and recenters on user
No resultsBottom sheet shows "No stores found in this area"
Navigation
Map View→Tap pin or store card→Store Detail
Map View→Back arrow→Origin screen
💡 The bottom sheet default "peek" state is important — users should immediately see 2–3 store cards without having to interact with it. Don't hide the list entirely behind the map on load.

Layout Specs
Context barStore name left · Availability summary pill right · purple-50 bg

Unavailable sectionGrouped card · 70% opacity · error-bg icon bg

Screen 4 — Store Confirmed

Success state · Payoff moment · Bridges back to origin screen

Components Used
Top Bar — Logo + Bell + Menu (no back) Confirm Hero (grad-purple + check circle) Store Summary Card Next Steps Box Button — Continue (primary) Button — Get Directions (ghost)
Hero Specs
Check circle64px · rgba(255,255,255,0.2) bg · 28px ✓ · Scales in on load
AnimationCheck scales 0→1.1→1 at 400ms spring · Hero fades up
No back arrowIntentional — user should not go back to Store Detail from here
Top barReturns to standard logo + icons (signals flow is complete)
Store Summary Card Specs
Backgroundpurple-50 · purple-100 border · r-xl
Section title11px · 700 · uppercase · purple-500
Detail rowsIcon + label + value · Right-aligned value · purple-100 divider
Data shownStore name · Address · Distance · Hours today
Next Steps Box Specs
Backgroundgold-50 · gold-100 border
PurposeTells user what to do next — reduces "now what?" confusion
ContentCoupon activated · How to pickup · Can change store later
Navigation — Context-Dependent "Continue"
Store Confirmed→Continue (from Onboarding)→Address Entry
Store Confirmed→Continue (from Home Pre)→Home Pre-Subscription
Store Confirmed→Continue (from Sub Registration)→Subscription Registration
Store Confirmed→Get Directions→External maps app
💡 "Continue" destination is determined by the entry point that triggered the Map flow. The origin must be passed as a parameter when the Map flow is launched so this screen knows where to return.
